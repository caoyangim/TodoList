# TodoFlow 业务规则

## 0. 账号注销

- 普通用户可从账号设置页永久注销自己的账号。
- 注销会删除该用户全部 Session、Todo、SOP 模板、执行实例、节点、附件元数据和
  磁盘附件文件，且不可恢复。
- 注销只影响当前用户，不影响管理员或其他用户的数据。
- 管理员账号不能注销，以保证系统始终保留管理入口。

## 1. Todo

- 标题必填，最长 200 字符。
- 时间优先级与重要优先级都为 `LOW`、`MEDIUM` 或 `HIGH`，默认 `MEDIUM`。
- Todo 状态分为 `PENDING`、`RESOLVED`、`COMPLETED`，页面文案分别为“待处理”“已解决（待验证）”“已完成（已验证）”。
- 默认状态为 `PENDING`。
- 只允许按流程流转：`PENDING → RESOLVED → COMPLETED`。
- 可从 `RESOLVED` 退回 `PENDING`，也可从 `COMPLETED` 退回 `RESOLVED` 或 `PENDING`。
- `completedAt` 仅在 `COMPLETED` 时写入，退回到其他状态时清空。
- 从“已解决”标记为“已完成”时，可选填写验证报告。
- 当前支持待处理、已解决、已完成和全部四个列表视图。
- 每个 Todo 可保存独立备注，备注复用 SOP 节点的富文本、链接和图片能力。
- 更新备注不改变 Todo 状态。
- 验证报告与备注一样使用富文本、链接白名单和附件能力，但只属于 Todo 的验证完成阶段。
- 每个 Todo 最多绑定一个 SOP 执行，Todo 与执行的完成状态互不自动修改。
- 删除 Todo 时保留绑定的执行；删除执行时保留 Todo 并解除绑定。

## 2. SOP 模板

### 基础规则

- 模板名称必填，最长 100 字符。
- 模板至少包含一个节点。
- 节点名称必填，最长 100 字符。
- 同一模板内 `sortOrder` 唯一。
- 已产生执行实例的模板不能删除。
- 更新模板不会修改历史执行实例。
- 管理员创建用户时，将自己名下的 `APP 发布流程` 和 `常规工作流` 深拷贝给
  新用户；模板及节点 ID 独立，用户可自行修改或删除。
- 默认模板只在创建用户时复制，管理员之后的修改不影响已创建的用户副本。

### 父子结构

- 当前仅支持两层：顶层节点与直接子节点。
- 子节点不能继续拥有子节点。
- `parentId = null` 表示顶层节点。
- `parentId` 必须指向同一模板中的顶层节点。
- 删除父节点时，表单中同时删除其子节点。
- 同级节点可通过上移和下移调整顺序。

### 必选与可选

- 只有叶子节点具有实际的必选/可选意义。
- 父节点成为分组节点后，完成状态完全由子节点聚合。
- 可选叶子节点不阻止整个执行实例完成。

### 备注必填

- 模板节点可设置 `noteRequired`（备注必填）开关，默认为关闭。
- 仅叶子节点可开启备注必填；父节点不显示此选项。
- 开启备注必填后，执行时该节点必须先填写备注（富文本正文或附件）才能勾选完成。
- 备注必填仅约束完成操作，不阻止撤销完成。
- 该设置属于模板快照，执行实例创建时一并复制，之后独立于模板。

## 3. SOP 执行快照

创建执行实例时必须在一个事务中：

1. 校验模板存在。
2. 校验模板至少有一个节点。
3. 规范化可选版本号；空字符串按 `null` 保存。
5. 复制模板名称和说明。
6. 复制所有节点字段。
7. 将模板节点 ID 映射为新的执行节点 ID。
8. 使用新 ID 重建 `parentId` 关系。

创建执行时可直接提交一个自定义模板。服务端必须在同一事务中创建模板、执行快照，
并在提供 `todoId` 时完成 Todo 绑定；任一步失败不得保留部分数据。

每个执行实例还具有独立标题：

- 标题必填，最长 100 个字符。
- 版本号为可选，最长 50 个字符。
- 标题不属于模板快照，可在执行详情中修改。
- 标题修改不改变模板名称快照、版本号、节点或完成状态。

执行实例创建后不再读取模板节点内容。

## 4. 节点完成规则

### 叶子节点

- 只有叶子节点可以手动勾选。
- 如果节点 `noteRequired` 为 true，必须先填写备注（含正文或附件）才能勾选完成。
- 重复提交相同状态是幂等操作，不更新时间。
- 从完成改为未完成前，前端必须显示确认弹窗。

时间字段：

| 字段 | 含义 |
|---|---|
| `completedAt` | 当前完成时间；撤销后清空 |
| `firstCompletedAt` | 首次完成时间；写入后永久保留 |
| `lastModifiedAt` | 最近一次勾选或撤销时间 |

显示规则：

- 有首次完成时间时显示“首次完成”。
- 上次修改与首次完成相同时不显示“上次修改”。
- 两者不同时同时显示。

### 父节点

- 父节点不能手动完成，API 会返回 `PARENT_NODE_READ_ONLY`。
- 父节点左侧完成按钮必须显示为禁用灰色。
- 点击父节点行只展开或收起子节点。
- 全部直接子节点完成时自动完成父节点。
- 任一子节点撤销时自动撤销父节点。
- 父节点同样保存首次完成与上次修改时间。

## 5. 执行状态与进度

进度只统计完成目标节点：

```text
存在必选叶子节点 → 已完成必选叶子节点数 / 全部必选叶子节点数
没有必选叶子节点 → 已完成叶子节点数 / 全部叶子节点数
```

完成目标：

```text
存在必选叶子节点 → 所有必选叶子节点完成
没有必选叶子节点 → 所有叶子节点完成
```

因此，存在未完成可选节点时，执行状态和进度都可以是已完成 / 100%，页面提示
“所有必须节点已完成”。

状态推导：

```text
completedAt 有值 → COMPLETED
startedAt 有值   → IN_PROGRESS
其他             → NOT_STARTED
```

## 6. 时间规则

- 数据库存储 ISO 8601 UTC 字符串。
- API 原样返回 UTC 字符串。
- 页面按浏览器本地时区格式化。
- 客户端不能指定完成时间。
- 服务端负责所有完成、撤销和聚合时间。

## 7. 执行归档与删除

- 每个执行实例可独立归档和恢复，归档不改变节点完成状态。
- `archivedAt` 为空表示未归档，有值表示已归档。
- SOP 执行首页先按模板分类，再将模板下的执行分为未归档和已归档。
- 删除执行前必须由前端弹窗二次确认。
- 确认删除后永久删除执行实例，并级联删除其全部执行节点。
- 删除模板的限制只取决于当前是否仍存在执行实例；删除全部执行后模板可再次删除。

## 8. 执行节点备注

- 每个执行快照节点都可保存独立备注，包括父节点和叶子节点。
- 备注由最多 2000 个可见字符的富文本正文和最多 10 张图片组成。
- 富文本支持段落、加粗、斜体、下划线、删除线、列表、引用、代码和链接。
- 服务端必须使用标签和属性白名单清洗正文；链接仅允许 `http`、`https` 和
  `mailto` 协议，并在新窗口打开。
- 支持从剪贴板直接粘贴 PNG、JPEG、WebP 和 GIF 图片，单张不超过 5 MB。
- 正文和图片都为空时按 `null` 保存。
- 备注属于执行实例，不回写 SOP 模板，也不影响其他执行实例。
- 更新备注不改变节点完成状态、首次完成时间、上次修改时间或执行进度。
- 更新备注会更新节点和执行实例的 `updatedAt`。
- 后续新增的备注功能必须复用通用富备注 DTO、编辑器和图片接口。
