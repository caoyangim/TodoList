# TodoFlow 业务规则

## 1. Todo

- 标题必填，最长 200 字符。
- 优先级为 `LOW`、`MEDIUM` 或 `HIGH`，默认 `MEDIUM`。
- `completedAt` 为空表示未完成，有值表示完成。
- 完成和恢复时间由服务端生成。
- 当前支持未完成、已完成和全部三个列表视图。

## 2. SOP 模板

### 基础规则

- 模板名称必填，最长 100 字符。
- 模板至少包含一个节点。
- 节点名称必填，最长 100 字符。
- 同一模板内 `sortOrder` 唯一。
- 已产生执行实例的模板不能删除。
- 更新模板不会修改历史执行实例。

### 父子结构

- 当前仅支持两层：顶层节点与直接子节点。
- 子节点不能继续拥有子节点。
- `parentId = null` 表示顶层节点。
- `parentId` 必须指向同一模板中的顶层节点。
- 删除父节点时，表单中同时删除其子节点。
- 同级节点可通过上移和下移调整顺序。

### 必选与可选

- 只有叶子节点具有实际的必选/可选意义。
- 父节点成为分组节点后，完成状态完全由子节点聚合。
- 可选叶子节点不阻止整个执行实例完成。

## 3. SOP 执行快照

创建执行实例时必须在一个事务中：

1. 校验模板存在。
2. 校验模板至少有一个节点。
3. 规范化并校验版本号。
4. 校验同一模板下版本号唯一。
5. 复制模板名称和说明。
6. 复制所有节点字段。
7. 将模板节点 ID 映射为新的执行节点 ID。
8. 使用新 ID 重建 `parentId` 关系。

执行实例创建后不再读取模板节点内容。

## 4. 节点完成规则

### 叶子节点

- 只有叶子节点可以手动勾选。
- 重复提交相同状态是幂等操作，不更新时间。
- 从完成改为未完成前，前端必须显示确认弹窗。

时间字段：

| 字段 | 含义 |
|---|---|
| `completedAt` | 当前完成时间；撤销后清空 |
| `firstCompletedAt` | 首次完成时间；写入后永久保留 |
| `lastModifiedAt` | 最近一次勾选或撤销时间 |

显示规则：

- 有首次完成时间时显示“首次完成”。
- 上次修改与首次完成相同时不显示“上次修改”。
- 两者不同时同时显示。

### 父节点

- 父节点不能手动完成，API 会返回 `PARENT_NODE_READ_ONLY`。
- 父节点左侧完成按钮必须显示为禁用灰色。
- 点击父节点行只展开或收起子节点。
- 全部直接子节点完成时自动完成父节点。
- 任一子节点撤销时自动撤销父节点。
- 父节点同样保存首次完成与上次修改时间。

## 5. 执行状态与进度

只统计叶子节点：

```text
progressPercent = 已完成叶子节点数 / 全部叶子节点数
```

完成目标：

```text
存在必选叶子节点 → 所有必选叶子节点完成
没有必选叶子节点 → 所有叶子节点完成
```

因此，存在未完成可选节点时：

- 执行状态可以是 `COMPLETED`。
- 进度可以低于 100%。
- 页面提示“所有必须节点已完成”。

状态推导：

```text
completedAt 有值 → COMPLETED
startedAt 有值   → IN_PROGRESS
其他             → NOT_STARTED
```

## 6. 时间规则

- 数据库存储 ISO 8601 UTC 字符串。
- API 原样返回 UTC 字符串。
- 页面按浏览器本地时区格式化。
- 客户端不能指定完成时间。
- 服务端负责所有完成、撤销和聚合时间。
