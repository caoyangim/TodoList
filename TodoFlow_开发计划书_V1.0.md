# TodoFlow 开发计划书

> **历史文档提示：** 本文件记录项目早期规划，部分技术方案和功能范围已与当前
> 实现不同。维护和开发请优先阅读 `AGENTS.md`、`README.md` 和 `docs/`。

> 个人使用的本地 Todo 与版本化 SOP 工具  
> 文档版本：V1.0  
> 更新日期：2026-06-13  
> 开发方式：小步迭代，每个迭代均可独立运行和验收

## 1. 产品目标

TodoFlow 解决两个问题：

1. 记录和完成日常 Todo。
2. 将重复工作保存为 SOP 模板，并基于模板创建互不影响的执行记录。

首个可用版本只打通以下流程：

```text
创建 Todo → 完成 Todo

创建 SOP 模板 → 创建执行实例 → 勾选节点 → 完成执行
```

## 2. 开发原则

- 先完成可使用的最小闭环，再增加体验功能。
- 每次只开发一个可独立验收的小功能。
- 不为尚未出现的需求预建复杂架构。
- 核心业务逻辑与页面代码适度分离，方便后续复用。
- 数据结构保留必要扩展空间，但不提前实现扩展功能。
- 每个迭代结束后实际使用，再决定下一步。

## 3. V1 核心范围

### 3.1 Todo

- 新增、编辑和删除。
- 标题、说明、优先级和截止时间。
- 完成和恢复。
- 查看全部、未完成和已完成 Todo。

### 3.2 SOP 模板

- 新增、编辑和删除模板。
- 新增、编辑、删除和排序节点。
- 模板至少包含一个节点。
- 已产生执行实例的模板不能删除。

### 3.3 SOP 执行

- 选择模板并输入版本号创建执行实例。
- 创建时复制模板和节点快照。
- 勾选或撤销节点。
- 记录节点完成时间。
- 显示执行进度。
- 全部节点完成后自动标记完成。

### 3.4 本地数据

- 使用 SQLite 保存数据。
- 应用重启后数据不丢失。
- README 说明数据库文件位置和手动备份方法。

## 4. 暂不开发

- 登录、权限和多人协作。
- 工作台统计和图表。
- Todo 搜索、复杂筛选、分页、标签和重复任务。
- 模板分类、复制、启用和停用。
- 可选节点、节点依赖、审批和备注。
- 执行取消、负责人和操作日志。
- 数据导入、导出和网页备份恢复。
- 消息通知和外部系统集成。
- 局域网和公网部署。
- 移动端专项适配。

这些功能不进入当前排期。只有实际使用产生明确需求时，才创建新的小迭代。

## 5. 技术方案

### 5.1 技术栈

| 用途 | 技术 |
|---|---|
| 应用 | Next.js + TypeScript |
| 样式 | Tailwind CSS |
| 数据校验 | Zod |
| 数据访问 | Prisma |
| 数据库 | SQLite |
| 测试 | Vitest |
| 代码检查 | ESLint + Prettier |

首版不引入 TanStack Query、React Hook Form、dnd-kit、状态管理库和通用组件库。

### 5.2 应用结构

```text
src/
├─ app/
│  ├─ todos/
│  ├─ templates/
│  └─ runs/
├─ actions/
│  ├─ todo.ts
│  ├─ template.ts
│  └─ run.ts
├─ components/
└─ lib/
   ├─ db.ts
   ├─ validations.ts
   └─ run-status.ts
prisma/
├─ schema.prisma
└─ migrations/
tests/
```

约定：

- 页面负责展示和接收输入。
- Server Actions 负责写操作和页面刷新。
- `lib` 只存放数据库连接、校验和可独立测试的业务函数。
- 文件出现明显重复或过大时再拆分，不预建 repository、service、mapper 等层级。

### 5.3 扩展边界

为后续迭代保留以下边界即可：

- 所有写入通过 Server Actions，未来需要外部调用时再增加 API。
- 输入使用 Zod schema，页面和服务端可复用规则。
- 执行状态计算放入纯函数，便于增加新状态。
- 数据库修改统一使用 Prisma migration。
- 执行实例保存快照，模板未来扩展时不会破坏历史数据。

## 6. 数据模型

只建立 5 张业务表。

### 6.1 Todo

```text
id
title
description?
priority       LOW | MEDIUM | HIGH
dueAt?
completedAt?
createdAt
updatedAt
```

是否完成直接通过 `completedAt` 判断，不单独保存状态字段。

### 6.2 SopTemplate

```text
id
name
description?
createdAt
updatedAt
```

### 6.3 SopTemplateNode

```text
id
templateId
name
description?
sortOrder
createdAt
updatedAt
```

约束：`templateId + sortOrder` 唯一。

### 6.4 SopRun

```text
id
templateId
templateNameSnapshot
templateDescriptionSnapshot?
version
startedAt?
completedAt?
createdAt
updatedAt
```

约束：`templateId + version` 唯一。

状态不单独入库，通过时间计算：

```text
completedAt 有值      → COMPLETED
startedAt 有值        → IN_PROGRESS
其他                  → NOT_STARTED
```

### 6.5 SopRunNode

```text
id
runId
nameSnapshot
descriptionSnapshot?
sortOrder
completedAt?
createdAt
updatedAt
```

约束：`runId + sortOrder` 唯一。

## 7. 核心业务规则

### 7.1 创建执行实例

必须在一个数据库事务内：

1. 确认模板存在且至少有一个节点。
2. 规范化版本号并检查是否重复。
3. 创建执行实例并保存模板快照。
4. 按顺序复制所有节点快照。

### 7.2 勾选节点

1. 完成时间由服务端生成。
2. 重复勾选已完成节点不更新时间。
3. 首个节点完成时写入实例 `startedAt`。
4. 全部节点完成时写入实例 `completedAt`。

### 7.3 撤销节点

1. 清空节点 `completedAt`。
2. 撤销后仍有已完成节点，实例保持进行中。
3. 所有节点均未完成时，清空实例 `startedAt`。
4. 撤销已完成实例中的任一节点时，清空实例 `completedAt`。

### 7.4 模板快照

- 模板修改只影响之后创建的实例。
- 历史实例只读取自己的快照。
- 已有执行实例的模板不能删除。

## 8. 页面

只开发 5 个页面：

| 页面 | 核心功能 |
|---|---|
| `/todos` | Todo 列表、新增、编辑、完成、删除 |
| `/templates` | 模板列表、新增、编辑、删除 |
| `/templates/:id` | 编辑模板及节点 |
| `/runs` | 执行实例列表和创建入口 |
| `/runs/:id` | 查看进度、勾选和撤销节点 |

根路径 `/` 直接跳转到 `/todos`，不开发独立工作台。

节点排序首版使用上移、下移按钮，不做拖拽。

## 9. 小步迭代计划

每个迭代完成后都应达到：

- 应用可以启动。
- 本次功能可以完整操作。
- 数据可以正确保存。
- 类型检查和相关测试通过。

### 迭代 0：可运行骨架

预计 0.5–1 天。

- 初始化项目和数据库。
- 建立基础导航和页面布局。
- 配置 Prisma migration、Zod、Vitest 和代码检查。

验收：打开应用可以在 Todo、模板和执行页面之间导航。

### 迭代 1：最小 Todo

预计 1–2 天。

- 创建 Todo。
- 展示 Todo 列表。
- 完成、恢复和删除 Todo。
- 按未完成、已完成切换查看。

验收：可以在日常使用中替代一张简单 Todo 清单。

### 迭代 2：SOP 模板

预计 1–2 天。

- 创建和编辑模板。
- 添加、编辑、删除节点。
- 使用按钮调整节点顺序。

验收：可以保存一个真实可用的 SOP。

### 迭代 3：SOP 执行

预计 2–3 天。

- 基于模板创建版本化执行实例。
- 保存模板和节点快照。
- 勾选、撤销节点并显示进度。
- 自动计算开始和完成状态。

验收：可以完整执行一次真实 SOP，且修改模板不会影响历史实例。

### 迭代 4：补齐 Todo

预计 1 天。

- 编辑标题和说明。
- 设置优先级和截止时间。
- 优化必要的表单校验和错误提示。

验收：Todo 核心字段完整可用。

### 迭代 5：首版收尾

预计 0.5–1 天。

- 修复实际使用中发现的阻塞问题。
- 验证重启后的数据持久化。
- 编写启动、数据位置和手动备份说明。

验收：在本机连续使用一轮核心流程无阻塞。

## 10. 测试范围

只自动测试容易出错的核心规则：

- Todo 完成和恢复。
- 同一模板下版本号不能重复。
- 创建执行实例时完整保存快照。
- 模板修改不影响历史实例。
- 节点勾选和撤销后的时间、进度与状态。

页面交互首版采用核心流程手工验收，不设置覆盖率目标，不建立完整端到端测试体系。

## 11. 首版完成标准

- [ ] Todo 可以新增、编辑、删除、完成和恢复。
- [ ] Todo 支持优先级和截止时间。
- [ ] SOP 模板及节点可以维护和排序。
- [ ] 可以根据模板创建唯一版本的执行实例。
- [ ] 执行实例保存独立快照。
- [ ] 节点完成时间、进度和状态计算正确。
- [ ] 修改模板不会影响历史实例。
- [ ] 应用重启后数据保持。
- [ ] README 包含启动和手动备份说明。
- [ ] 核心业务测试通过。

## 12. 后续迭代规则

每次只从实际使用问题中选择一个主题，例如：

- Todo 搜索和筛选。
- 模板复制。
- 节点备注。
- 操作历史。
- 数据备份界面。
- Docker 部署。

新增功能应遵循：

1. 先记录使用场景和最小验收条件。
2. 一次迭代只解决一个主题。
3. 优先复用现有模型和操作入口。
4. 完成并实际使用后，再决定下一个迭代。
